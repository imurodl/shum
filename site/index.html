shum

A trust-first CLI for safer Docker Compose upgrades on Linux hosts.

What shum protects

Self-hosted environments break in production windows. shum starts from explicit trust and explicit state:

  • SSH aliases with known-host verification
  • Runtime-first project discovery
  • Canonical context checks before changing state
  • Stored inspect artifacts for evidence and rollback planning
  • Deterministic upgrade plans with preflight gating
  • Backup and restore hooks + rollback workflow

Quickstart

Register a host, discover projects, inspect, then run safely:

shum host register prod
shum project discover prod
shum project inspect prod web --project-directory /srv/web --json
shum project preflight prod web
shum project plan prod web --json

Inspect history for a specific target:

shum project run list --host prod --project web --json

Set policy, back up, and upgrade:

shum project policy set prod web \
  --require-backup=true \
  --backup-command "docker exec web pg_dumpall -U app > \"$SHUM_BACKUP_ARTIFACT\"" \
  --restore-command "docker exec -i web psql -U app < \"$SHUM_BACKUP_ARTIFACT\""
shum project upgrade prod web --dry-run
shum project run list --limit 10 --json
shum project run show run-1700000000000000000 --json
